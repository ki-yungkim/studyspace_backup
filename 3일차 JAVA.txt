##2차원 배열
-- 행과 열의 구조를 갖는 배열
-- 행마다 열의 크기를 동일하게 생성
-- 행마다 열의 크기를 다르게 생성  (메모리 효율적 관리)
-- 2차원배열명.length : 2차원 배열의 행크기 저장된 변수
-- 2차원배열명[행번호].length : 해당행의 열크기 저장된 변수
-- 행번호 시작 : 0
-- 열번호 시작 : 0
 
## 행마다 열의 크기를 동일하게 생성
1. 선언, 생성, 초기화
-- 선언
	[modifiers] 타임[] [] 2차원배열명; //권장
	[modifiers] 타임 2차원배열명[][];
	[modifiers] 타임[] 2차원배열명[]; //OCJT시험에나 나오는 문제

-- 생성
	2차원배열명 = new 타입 [행크기][열크기};
	
-- 초기화(배열요소 사용)
	2차원배열명[행번호][열번호] = 값;
	
2. 선언 + 생성, 초기화
-- 선언 + 생성
	[modifiers] 타입[][] 2차원배열명 = new 타입[행크기][열크기]
	
3. 선언 + 생성 + 초기화 
	[modifiers] 타입[][] 2차원배열명 = new 타입[행크기][열크기]{
	{value1, value2, value3 },
	{value1},
	{value1, value2, value3, value4}
}
	[modifiers] 타입[][] 2차원배열명 = {
	{value1, value2, value3 },
	{value1},
	{value1, value2, value3, value4}
}

## 자바 프로그래밍 컴파일 방법
-- default package 컴파일방법
dos> javac [options] *.java
dos> javac HelloWorld.java -> 인코딩 오류
javac -> 옵션, 사용법 알려줌
javac -encoding utf-8 HelloWorld.java
		=> 클래스이름.class (byte code)

-- package 컴파일방법		
dos> javac -d output-dir-name *.java
	
## 자바 프로그래밍 실행 방법
-- 자바실행 : .class 바이트코드의 확장자를 지정해서는 안됨(. 패키지구분자)

-- default package 실행방법
dos> java [options] main() 매서드가있는실행클래스명
dos> java HelloWorld

-- package 실행방법
dos> java [options] 패키지명.패키지명.main() 매서드가있는실행클래스명
dos> java HelloWorld

-- main() 매서드 아규먼트 전달 실행방법
dos> java [options] main()매서드가있는실행클래스명 아규먼트데이터1 "아규먼트 데이터x"
	=> jvm이 실행시에 클래스명 뒤에 공백을 구분자로 해서 해당 크기만큼의 String 타입의 배열객체를 생성해서 순서대로 배열요소 값을 할당
	
Run as - Configuration - 클래스 선택하고 - Arguments - 
옵션들이 VM 아규먼트
